# Admin guide — overview

The **admin panel** at [legaleaglelaws.com/admin](https://legaleaglelaws.com/admin) is the workspace for firm staff. It is gated by Google Sign-In plus role-based access control (admin, manager, editor, creator). This section of the documentation is being built out incrementally; this page is the table of contents for what is planned and what is shipped.

## Sections

| Section | Status |
|---|---|
| Sign-in, role hierarchy, feature permissions | Planned (next documentation batch) |
| Blog editor — drafts, scheduling, comments moderation | Planned |
| Plan management for the SaaS lawyer tier | Planned |
| User management — roles, feature toggles, suspensions | Planned |
| Calendar account — connecting the firm's Google Calendar for booking | Planned |
| Chatbot tuning — cache review, KB curation, rate limits | Planned |
| Content migration — importing from the legacy PHP site | Planned |
| Settings — firm details, contact info, brand assets | Planned |

If you are firm staff and need a specific admin page documented sooner, please [open an issue](https://github.com/aoneahsan/legaleaglelawfirm-docs/issues) on the documentation repository.

## In the meantime

The admin panel is self-explanatory for most workflows — every screen has inline copy explaining what the field does. The structures the admin panel manages are documented from the architecture side at:

- [Architecture overview](../architecture/intro.md)
- [User guide for clients](../user-guide/intro.md) — useful for understanding the client view of the data admin staff manage.
